# NedoBot

AI-помощник для Telegram-чата Nedonews на Rust и teloxide.

Бот следит за постами канала в привязанном discussion-чате, отличает обычные редакционные публикации от рекламы, генерирует первый комментарий через vision LLM, зовёт людей в чат живым CTA и собирает статистику реакции аудитории.

## Что Умеет

- Находит авто-форварды постов из канала в discussion-чате.
- Пропускает рекламные/служебные посты без редакционного маркера.
- Вырезает хвосты VK/MAX перед отправкой текста в модель.
- Передаёт в LLM текст, фото поста, RAG, память прошлых новостей и последние ответы бота.
- Генерирует первый комментарий под постом с HTML-ссылкой на чат и выключенным link preview.
- Использует custom/premium emoji канала и тематические emoji для AMD/Radeon/Ryzen.
- Ведёт автоматическую память новостей и объединяет похожие заметки.
- Собирает статистику чата, пользователей, реплаев, медиа, ссылок, реакций и активности после комментариев бота.
- Хранит события и сырые Telegram JSON в Postgres, чтобы можно было дообогащать данные позже.

## Быстрый Старт

1. Скопируй конфиг:

```bash
cp .env.example .env
```

2. Заполни токены и id чатов в `.env`.

3. Подними Postgres:

```bash
docker compose up -d postgres
```

4. Запусти бота:

```bash
cargo run
```

Проверка сборки:

```bash
cargo check
```

## Конфиг

Основные переменные:

```env
TELOXIDE_TOKEN=
DATABASE_URL=postgres://tg_ai_bot:tg_ai_bot@localhost:5432/tg_ai_bot

SOURCE_CHANNEL_ID=
DISCUSSION_CHAT_ID=
CHAT_INVITE_URL=
CHAT_INVITE_LABEL=чате
POST_SIGNATURE_MARKER=Не теряем связь

OLLAMA_API_KEY=
OLLAMA_BASE_URL=https://ollama.com
VISION_MODEL=gemma4:31b
```

Локальные `.env` файлы не коммитятся.

## Как Работает Первый Комментарий

1. Telegram присылает авто-форвард поста из канала.
2. Бот проверяет source channel и наличие редакционного маркера.
3. Из поста удаляется хвост `Не теряем связь — VK / MAX`.
4. Самое большое фото поста уходит в vision-модель.
5. В prompt добавляются:
   - текст поста;
   - короткий tech RAG;
   - релевантные заметки памяти;
   - последние комментарии бота для анти-повтора;
   - реальное число участников чата, но модель просится использовать его редко.
6. Модель возвращает текст с `{CHAT_LINK}`.
7. Код сам заменяет плейсхолдер на HTML-ссылку и отправляет reply под постом.

## Статистика

Периоды считаются по московскому времени:

- день: с 05:00;
- неделя: с понедельника 05:00;
- месяц: с 1 числа 05:00.

Собирается:

- общее число сообщений;
- активные пользователи;
- реплаи, ссылки, медиа;
- посты канала и комментарии бота;
- реплаи на комментарии бота;
- reaction updates и reaction count updates;
- входы/выходы, если Telegram отдаёт `chat_member` updates;
- активность после комментария бота за 5 и 30 минут;
- расширенная карточка пользователя в `telegram_chat_users`.

Имена пользователей в отчётах показываются как кликабельные Telegram-ссылки без видимого ID.

## Ограничения

- Старые реакции Telegram Bot API не отдаёт задним числом. Их можно будет восстановить отдельно, если импортировать внешний JSON.
- Join/leave и реакционные события зависят от того, какие update-типы Telegram реально отдаёт боту.
- Админки пока нет: настройки через `.env` и рестарт сервиса.
