# TODO: гибридный поиск обсуждений для первого комментария

Статус: реализованы ingestion, единый research-план, shadow hybrid retrieval, структурная provenance и безопасный рендер. Production evidence остаётся выключенным до ручной оценки shadow-выборки и отдельной калибровки selection stage.

## Цель

Первый комментарий должен уметь:

- находить реальные похожие обсуждения в чате за последние 14–30 дней;
- загружать не одиночный match, а содержательную ветку либо соседний контекст;
- ссылаться на реальные сообщения и делать имена участников кликабельными;
- использовать найденное обсуждение как живое приглашение в чат;
- при отсутствии обсуждения шутливо приглашать поговорить о новости, не утверждая, что её уже обсуждают;
- не менять основную тему и аудиторию поста из-за второстепенного поискового результата.

## Подтверждённые решения

- Бот публикует первый комментарий, поэтому контекст ищется в прошлых обсуждениях всего чата, а не в новой ветке поста.
- Окно retrieval по умолчанию — 30 дней, с геометрически убывающим весом свежести.
- RuBERT используется совместно с полнотекстовым и точным поиском.
- Gemma формирует единый research-план сразу для чата, веба, Reddit и GitHub.
- Если найдено сообщение в ветке, в LLM-контекст передаётся ветка; для одиночного сообщения — ограниченный соседний контекст.
- Имена упомянутых участников должны быть ссылками.
- В одном комментарии разрешено упоминать не более трёх участников.
- Ссылка на участника или сообщение уже является ссылкой в чат; отдельный `CHAT_LINK` в таком комментарии не обязателен.
- Утверждение «это уже обсуждали в чате» — редкий high-confidence режим, а не обязательный результат retrieval.

## 3. Гибридный retrieval

- [x] Искать кандидатов за последние 30 дней по объединению:
  - максимальной cosine similarity по Gemma semantic-запросам;
  - PostgreSQL full-text `russian`;
  - PostgreSQL full-text `simple`;
  - безопасных literal-regex совпадений `chat_lexical_terms`.
- [ ] Для каждого латинского фрагмента детерминированно строить варианты:
  - исходный case-insensitive literal;
  - русский транслит;
  - нормализованный вариант со смешанными кириллическими и латинскими гомоглифами;
  - транслит после нормализации гомоглифов.
- [ ] Переиспользовать общую нормализацию гомоглифов; не заводить отдельную несовместимую таблицу замен.
- [ ] Экранировать все regex-метасимволы и передавать шаблоны параметрами SQL: текст из Gemma и поста не должен становиться исполняемым regex или частью SQL.
- [ ] Давать дополнительный вес буквальным совпадениям латинских терминов: `KMS`, `TPM`, `ROCm`, названия GPU, версии и команды.
- [ ] Учитывать свежесть по геометрически убывающей функции, а также длину содержательного текста и наличие ответов.
- [ ] Использовать настраиваемый half-life; стартовое предложение — 7 дней:

```text
freshness_score = 2 ^ (-age_days / 7)
```

При таком старте вес равен `1.0` сегодня, `0.5` через 7 дней, `0.25` через 14 дней и примерно `0.05` через 30 дней. Старое обсуждение всё ещё может победить при сильном semantic/exact совпадении и хорошей ветке.
- [ ] Исключать сообщения бота, channel forwards, спам и удалённые записи.
- [ ] Дедуплицировать совпадения, принадлежащие одной ветке.
- [ ] Ограничить число кандидатов до передачи в LLM.
- [ ] Подобрать пороги на реальных сообщениях и сохранять отдельные составляющие score, а не только итоговый балл.

Предлагаемый rank:

```text
semantic_score
+ lexical_score
+ exact_term_boost
+ thread_activity_boost
+ geometric_freshness_score
```

Критерий готовности:

- английский exact match не теряется из-за особенностей русскоязычного RuBERT;
- латинское слово находится при транслитерации и смешении кириллических/латинских гомоглифов;
- один активный тред не заполняет весь список одинаковыми сообщениями.

## 7. Два режима генерации комментария

### Есть подтверждённое обсуждение

- [ ] Разрешить живую формулировку о реальных участниках и прошлом разговоре.
- [ ] Требовать опору на выбранную ветку и ссылки на имена или сообщение.
- [ ] Не преувеличивать один комментарий до «весь чат спорит».
- [ ] Не утверждать, что старое обсуждение относится к текущему посту буквально; допустимо «обсуждали похожую проблему».
- [ ] Включать этот режим только при высоком итоговом score и связной ветке; слабое совпадение использовать лишь как внутренний контекст либо полностью игнорировать.
- [ ] Не пытаться находить прошлое обсуждение для каждого поста: отсутствие high-confidence evidence считается нормальным результатом.
- [ ] Выбирать для упоминания от одного до трёх наиболее содержательных участников ветки.

### Подтверждённого обсуждения нет

- [ ] Разрешить шутку, провокационный вопрос или игровой призыв к обсуждению.
- [ ] Запретить утверждения, что люди уже спорят, считают, обсуждают или ждут.
- [ ] Использовать `CHAT_LINK` как часть реплики, а не обязательный канцелярский хвост.

Критерий готовности:

- бот остаётся живым и смешным без выдуманной социальной активности;
- отсутствие retrieval-кандидатов не приводит к сухому шаблону.
- ссылки на прошлое обсуждение появляются редко, но точно соответствуют теме.

- [ ] KMS regression: consumer/piracy scope не заменяется server-admin scope.
- [ ] Точный `KMS`/`TPM` match получает lexical boost.
- [ ] Английская аббревиатура находится через `simple` и literal search.
- [ ] Латинский термин находится по русскому транслиту.

- [ ] Semantic-only русская перефразировка находится через RuBERT.
- [ ] Reply hit раскрывается в ограниченную ветку.
- [ ] Standalone hit получает соседний контекст.
- [ ] Несвязанные соседи не становятся доказательством общего обсуждения.
- [ ] Два автора дают право на формулировку о разговоре нескольких людей.
- [ ] Более трёх найденных авторов сокращаются до трёх упоминаний.
- [ ] Один автор не превращается во «все уже обсуждают».
- [ ] `CHAT_AUTHOR` использует public username, если он есть.
- [ ] `CHAT_AUTHOR` без username ведёт на сообщение.
- [ ] Неизвестный message ID отклоняется валидатором.
- [ ] Chat evidence заменяет обязательный `CHAT_LINK`.
- [ ] Без chat evidence `CHAT_LINK` остаётся обязательным.
- [ ] External search result вне primary scope можно не использовать.
- [ ] Слабый chat match не включает режим «уже обсуждали».
- [ ] Геометрический freshness rank соответствует выбранному half-life.
- [ ] Raw URL, HTML и выдуманные имена не проходят validation.
